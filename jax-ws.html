  2. purpose of <soapAction> attribute in WSDL
    3. In the WSDL omitting the soapAction attribute is equivalent to setting soapAction="". It means that the SOAPAction HTTP header will be empty. The WS-I Basic Profile requires that a SOAP envelope inside an HTTP POST request is accompanied by an HTTP SOAPAction header - however it permits the SOAPAction header to be empty. The soapAction attribute simply specifies the content of the SOAPAction HTTP header. The content of the SOAPAction HTTP header is supposed to communicate the "intent" (e.g. operation) of the SOAP envelope, so that proxies and firewalls can act on it without looking into the XML SOAP envelope.
  3. namespaces for WSDL versions
    • WSDL 1.1 → http://schemas.xmlsoap.org/wsdl/ (source)
    • WSDL 1.2 → http://www.w3.org/2002/07/wsdl (source)
  4. namespaces for SOAP versions
    • SOAP 1.2 → http://www.w3.org/2003/05/soap-envelope (source)
    • SOAP 1.1 → http://schemas.xmlsoap.org/soap/envelope/ (source)
