1. piping with xargs
    2. To rm all java files (you need the -print0 and -0 options to work with filenames containing spaces etc.): 
    find . -iname \*.txt -print0 | xargs -0 rm
  2. recursively copy with permissions
  3. cp -pr
  4. pipe coloured output through less
    5. Use the 'ls -RG' option and make sure the original application (that generates the colour-coded output) is set to --color=always if such option exists.
  5. find the package a binary belongs to in Debian distros

    6. E.g. find the package for sensors: 

    dpkg -S $(which sensors)
  6. monitor chipset and CPU sensors in Ubuntu
    7. Use sensors from package lm-sensors as in: 
    watch sensors
  7. change hostname in Ubuntu
    8. simply edit /etc/hostname and /etc/hosts (for 127.0.0.1). To flush the previous hostname from various applications a reboot may be required.
  8. setting HTTP proxy in Ubuntu
    9. export http_proxy="http://foo.bar:8080"
    subsequent wget requests will redirect to foo.bar:8080
  9. package managers
    10. apt is used in Debian, Ubuntu and rpm in Fedora, Redhat, Suse. apt is better.

    Typical apt commands: apt-get install, apt-cache search, apt-get autoremove, apt-get update.

  10. curl
    11. curl is a Linux utility that can be used to simulate HTTP GET and HTTP POST requests. A simple:
    #curl www.in.gr
    ...will execute an HTTP GET against www.in.gr.
  11. SSE time
    12. get time in SSE:
    date +%s
  12. Linux print system information
    13. including kernel 32 or 64 bit:
    uname -a
    x86_64 GNU/Linux indicates that you've a 64bit Linux kernel running. If you use see i386/i486/i586/i686 it is a 32 bit kernel.
  13. Find if CPU is 32 or 64 bits
    14. grep flags /proc/cpuinfo | grep lm

    ("lm" means long mode CPU - 64 bit CPU)
  14. find installed packages
    15. apt-cache search  - or -
dpkg --get-selections | grep
  15. install packages
    16. apt-get install
  16. how to find a file named "compile_decoder.sh" in a set of bz2 files
    17. for f in `ls *.bz2` ; do tar -jtvf  $f ; done | grep compile_decoder.sh

    ... Actually, the following also works and produces identical output: 
    for f in `ls *.bz2` ; do tar -jtvf $f | grep compile_decoder.sh ; done
  17. to find out the release
    18. $ lsb_release -a
